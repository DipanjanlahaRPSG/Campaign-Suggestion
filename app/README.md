# Promotional Content AI  📣

LLM-powered SMS/WhatsApp campaign content suggestion system.
Learns from your historical campaign data to generate optimised content variants.

---

## Project structure

```
promo_ai/
├── app.py                    ← Streamlit UI (entry point)
├── requirements.txt
├── .env.example              ← Copy to .env and add your OpenAI key
├── data/
│   ├── campaigns.csv         ← Your raw campaign data (place here or upload via UI)
│   ├── feature_store.csv     ← Generated by pipeline
│   └── feedback_log.csv      ← Auto-generated as you rate suggestions
└── src/
    ├── data_pipeline.py      ← Clean + derive metrics + assign tiers
    ├── benchmark_engine.py   ← Benchmark tables + top performer extraction
    └── ai_engine.py          ← OpenAI prompt builder + API caller + feedback logger
```

## Setup

### 1. Install dependencies
```bash
cd promo_ai
pip install -r requirements.txt
```

### 2. Add your OpenAI API key
Either:
- Copy `.env.example` → `.env` and fill in `OPENAI_API_KEY=sk-...`
- Or paste the key directly in the app sidebar (no file needed)

### 3. Run the app
```bash
streamlit run app.py
```

The app opens at `http://localhost:8501`

## How it works

### Data pipeline  (`src/data_pipeline.py`)
1. Loads your 47-column campaign CSV/Excel
2. Cleans nulls, standardises channel names, parses dates
3. Derives **Reach %** = Delivered / Sent and **Engagement %** = (Clicks + Opened_Read) / Delivered
4. Assigns a **performance tier** (top / mid / bottom) within Brand × Channel × Occasion groups

### Benchmark engine  (`src/benchmark_engine.py`)
- Computes mean, median, P25, P75 benchmarks per Brand × Channel × Occasion
- Extracts top-performing `Content_subject`, `Sentiment_v1/v2`, `Offer` examples
- Drives both the dashboard charts and the AI few-shot context

### AI engine  (`src/ai_engine.py`)
Prompt structure injected into GPT-4o:
```
Brand + Channel + Objective + Segment + Occasion
  ↓
Benchmark targets (reach_mean, engagement_p75, top_sentiment)
  ↓
Top 3-5 historical performers (content + metrics as few-shot examples)
  ↓
GPT-4o → 3 content variants with:
  - message text
  - sentiment + tone keywords
  - promo code / offer flags
  - predicted tier (top/mid/bottom)
  - rationale
```

### Feedback loop  (`data/feedback_log.csv`)
- Every 👍/👎 in the UI writes a row to `feedback_log.csv`
- After campaigns run, enter actual Reach % and Engagement % in the Feedback tab
- In Phase 2 this log will be used to re-rank content patterns and fine-tune prompts

## Key business metrics

| Metric | Formula | Source columns |
|---|---|---|
| Reach % | Delivered / Sent × 100 | Delivered, Sent |
| Engagement % | (Clicks + Opened_Read) / Delivered × 100 | Clicks, Opened_Read, Delivered |
| Bounce rate | (Hard + Soft Bounce) / Sent × 100 | Hard Bounce, Soft Bounce, Sent |
| Performance score | Reach × 0.4 + Engagement × 0.6 | derived |

## Tips for best results

- **More top-performing campaigns = better suggestions.** The AI uses your actual top performers as few-shot examples. The richer your data, the better the output.
- **Use the Occasion filter.** Festive campaigns perform very differently from winback campaigns — always filter when possible.
- **Log feedback consistently.** The feedback CSV is the foundation of Phase 2 prompt improvement.
- **Paste actual results.** After each campaign runs, go to the Feedback tab and log real Reach/Engagement — this closes the learning loop.
